# 📂 Documentación del Sistema: Requerimientos y Reglas de Negocio

Bienvenido a la carpeta de especificaciones del **Módulo Material Resources (MR)**. 

Este directorio contiene la documentación técnica detallada que define el alcance, el comportamiento esperado y las normativas operativas de la base de datos y el sistema que estamos diseñando para la **Plataforma Administrativa Hospitalaria**.

---

## 📑 Contenido del Directorio

En esta carpeta encontrarás los siguientes documentos en formato PDF, separados por categoría para facilitar su lectura y validación:

* 📄 **`[Requerimientos Funcionales].pdf` - Requerimientos Funcionales (FRs)**
    * **Descripción:** Detalla *qué* debe hacer el sistema. Incluye las operaciones específicas, procesos de gestión de inventario, proveedores, espacios y control de accesos que la base de datos debe soportar (CRUD y transacciones).

* 📄 **`[Requerimientos No Funcionales].pdf` - Requerimientos No Funcionales (NFRs)**
    * **Descripción:** Define *cómo* debe comportarse el sistema. Abarca los criterios de calidad, seguridad, rendimiento, escalabilidad y las normas de estandarización necesarias para garantizar la correcta integración con otros módulos del hospital.

* 📄 **`[Reglas de Negocio].pdf` - Reglas de Negocio (BRs)**
    * **Descripción:** Establece las *políticas y restricciones* propias de la administración del hospital. Estas son las normativas lógicas e inquebrantables que el diseño de la base de datos debe hacer cumplir obligatoriamente (ej. un equipo en mantenimiento no puede asignarse a un área clínica).
